=====================================Модуль Регион====================================
/api/region/region - создание региона (права на создание имеют только администраторы
/api/region/region/id - изменение/удаление текущего региона
/api/region/city - создание города (при создании обязательно указывать регион)
/api/region/city/2 - изменение/удаление текущего города


=====================================Модуль Пользователей====================================
/api/users/admin - создание профиля админисnратора (при создании обязательно указать id пользователя)
/api/users/admin/id - изменение/удаление текущего профиля администратора
/api/users/rp - создание профиля регионального представителя (при создании обязательно указать id пользователя)
/api/users/rp/id - изменение/удаление текущего профиля регионального представителя
# Для регионального представителя будет выводится только его профиль
/api/users/executor - создание профиля исполнителя (при создании обязательно указать id пользователя)
/api/users/executor/id - изменение/удаление текущего профиля исполнителя
# Для регионального представителя будет выводиться список всех его исполнителей
/api/users/customer - создание профиля заказчика (при создании обязательно указать id пользователя)
/api/users/customer/id - изменение/удаление текущего профиля заказчика
/api/users/password/id - изменеие пароля пользователя (id - идентификационный номер пользователя)
/api/users/loin - вход в систему пользователя (возвращает токен, который должен использоваться в загаловках запроса для  авторизации)
/api/users/lock-executor/id - юлокирует текущего пользователя (исполнителя) 
/api/users/lock-rp/id - блокировка текущего регионального представителя (все не архивные заявки где фигурирует данный РП заменяются на его зама)


=====================================Модуль Пасспорт======================================
/api/passport - создание пасспорта пользователя (при создании необходимо указать id пользователя)
/api/passport/id - изменение/удаление текущего пасспорта пользователя

Для каждого пользователя будет выводиться только его пасспорт, изменять он может только свой паспорт


=====================================Модуль Кошелек========================================
/api/wallet - создание кошелька пользователя (при создании необходимо указать id пользователя)
/api/wallet/id - изменение/удаление текущего кошелька пользователя

Для каждого пользователя будет выводиться только его кошедек, изменять он может только свой кошелек (за исключением балансов)
Для каждого пользоавтеля автоматически создается кошелек с нулевым балансом при создании пользователя
Для системного кошелька (на который происходит списание комиссии) необходимо ручное создание с указанием id пользователя равным 1 (системный администратор или 
первое зарегистрированное лицо в приложении)


=====================================Модуль Рейтинг========================================
/api/rating - моделm рейтинга для польозвателя (исполнитель)

# Доступно только действия просмотра. Если рейтинг для данного пользователя существует, то он отобразится, если нет, то
# будет выведена пустая строка


=====================================Модуль Уведомления====================================
/api/notification - создание уведомления (обязательно указывать id заявки)
/api/notification/id - изменение/удаление текущего уведомления (только изменение статуса с непрочитано (2) на прочитано (1))

Параметры запроса:
order = id - id заявки

# Поле type определяется автоматически по пользователю, осуществляющего запрос
# Выводятся только непрочинные уведомления


=====================================Модуль Заявки=========================================
/api/order/list - список всех активных заявок

Параметры запроса:
region = id  - id региона (обязательный параметра запроса)
status = number - статус заявки
approve = number - одобрена заявка или нет после добавления и проверки отчет по заявке региональным представителем
1 - одобрена, 2 - не одобрена

Возможно 3 варианта фильтрации заявок:
1. только по региону
2. по региону и по статусу
3. по региону, по статусу, по полю одобрен отчет или нет

/api/order/create - создание заявки
/api/order/create/id - изменение/удаление текущей заявки
/api/order/execute/id - добавление исполнителя в поле исполнитель текущей заявки
/api/order/accept/id - исполнитель подтвердил правильность указанной заказчиком информации в заявке
/api/order/decline/id - исполнитель отклонил заявку (создается уведомление заказчику о неверности указанной информации) 
/api/order/not_valid_list - отображение всех заявок, которые вернулись заказчику по причине некорректно указанной информации
/api/order/not_valid_list/id - исправление информации в некорректно заполненной заявке

/api/order/reviews - просмотр, создание отчета (при создании автоматически создается уведомление региональному представителю о том что в текущую заявку добавлен отчет)
/api/order/reviews/id - изменение/удаление текущего отчета (при создании автоматически создается уведомление региональному представителю о том что в текущей заявку изменен отчет)

Параметры:
order = id - Идентификационный номер пользователя, создавшего отчет.

/api/order/review-images - просмотре всех изображений отчета
/api/order/review-images/id - изменение/удаление изображения
/api/order/review-scans - просмотре всех сканов отчета
/api/order/review-scans/id - изменение/удаление скана

Параметры запроса для модулей review-images и review-scans:
review = id - id отчета (обязательный параметр)

/api/order/review-accept/id - одобрение региональным представителем отчета по заявке. Автоматически переводит заявку в закрытое состояние и производит выплаты
/api/order/review-decline/id - отклонение региональным представителем отчета по заявке. Автомически создается уведомление для исполнителя о необходимости исправить отчет.

# Данные действия может осуществлять только региональный представитель
